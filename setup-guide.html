TheLifeCo App — Production Setup Guide

Step-by-step account creation and configuration for thelifeco.app

1
Cloudflare — DNS & Email Routing

Cloudflare will manage your DNS records and route emails to thelifeco.app addresses.

1.1 Create Cloudflare Account Free

  1. Go to Cloudflare and create an account
  2. Use your personal email (you'll change this later if needed)
  3. Click "Add a site" and enter thelifeco.app
  4. Select the Free plan
  5. Cloudflare will scan existing DNS records — keep them for now
Sign up for Cloudflare

1.2 Update Domain Nameservers

Cloudflare will give you two nameservers. Update them at your domain registrar (wherever you bought thelifeco.app).

Where to update: Go to your domain registrar (e.g., Namecheap, Google Domains, GoDaddy) → find thelifeco.app → change nameservers to the ones Cloudflare gives you. This can take up to 24 hours to propagate.

1.3 Set Up Email Routing Free

  1. In Cloudflare dashboard, go to thelifeco.app → Email → Email Routing
  2. Click Get started
  3. Add a Custom address: admin@thelifeco.app
  4. Set the Destination to your personal email (where you want to receive mail)
  5. Cloudflare will send a verification email to your personal email — click the link
  6. Repeat to add: bookings@thelifeco.app → same personal email
How it works: Any email sent to admin@thelifeco.app will appear in your personal inbox. You're not creating a new mailbox — Cloudflare just forwards the mail.
Open Email Routing

2
Resend — Transactional Email API

Resend lets the app send emails programmatically (welcome messages, booking confirmations, etc.).

2.1 Create Resend Account Free — 100 emails/day

  1. Go to Resend and sign up (use your personal email or admin@thelifeco.app if it's routing already)
  2. Once logged in, go to Domains → Add Domain
  3. Enter: thelifeco.app
  4. Resend will show you DNS records to add
Sign up for Resend

2.2 Add DNS Records in Cloudflare

Resend will give you these records to add. Go to Cloudflare → DNS → Records and add them:

TypeNameValue
TXTresend._domainkey(Resend will provide this — it's a DKIM key)
TXTthelifeco.appUpdate SPF: v=spf1 include:_spf.mx.cloudflare.net include:amazonses.com ~all
Important: Don't create a duplicate SPF record. Edit the existing TXT/SPF record to add include:amazonses.com before the ~all.

2.3 Create API Key

  1. In Resend dashboard, go to API Keys
  2. Click Create API Key
  3. Name it: thelifeco-app
  4. Permission: Full access
  5. Copy the key immediately — you won't see it again
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxxx
Save this key! You'll add it as a Vercel environment variable in Phase 4.
Manage API Keys

2.4 Verify Domain

  1. Back in Resend → Domains, click Verify next to thelifeco.app
  2. Wait for DNS propagation (usually 5-30 minutes)
  3. Status should change to Verified
Check Domain Status
  • Resend account created
  • thelifeco.app domain added in Resend
  • DNS records added in Cloudflare
  • Domain verified in Resend
  • API key created and saved securely
3
Supabase — Database & Authentication

Supabase provides the PostgreSQL database, user authentication, and file storage.

3.1 Create Supabase Account Free — 500MB DB, 1GB storage

  1. Go to Supabase and sign up with your GitHub account (recommended) or email
  2. If using email, use admin@thelifeco.app
Sign up for Supabase

3.2 Create New Project

  1. Click New Project
  2. Organization: Create one called "TheLifeCo"
  3. Project name: thelifeco-portal
  4. Database password: Generate a strong password and save it!
  5. Region: Choose East US (N. Virginia) — closest to St. Lucia
  6. Click Create new project — takes 1-2 minutes

3.3 Get Your API Keys Required

  1. Go to Settings → API
  2. Copy these three values:
SUPABASE_URL=https://xxxxxxxxxx.supabase.co SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs... (long key) SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIs... (different long key)
Security: The SERVICE_ROLE_KEY bypasses all security rules. Never expose it in client-side code. It's only used in Vercel serverless functions.
Open API Settings

3.4 Run Database Schema

  1. Go to SQL Editor in Supabase dashboard
  2. Click New query
  3. Open the file supabase/schema.sql from your project
  4. Paste the entire contents into the SQL editor
  5. Click Run — this creates all 15 tables
  6. You should see "Success. No rows returned" for each statement
Open SQL Editor

3.5 Run Seed Data

  1. Create another New query in SQL Editor
  2. Open the file supabase/seed.sql from your project
  3. Paste and Run
  4. This loads all therapy data, Hasan's schedule, hydration items, etc.

3.6 Create Admin User

  1. Go to Authentication → Users
  2. Click Add user → Create new user
  3. Email: admin@thelifeco.app
  4. Password: Choose a strong password
  5. Check Auto Confirm User
  6. Click Create user
  7. Copy the user's UUID from the users table
  8. Go back to SQL Editor and run:
INSERT INTO staff (id, display_name, role) VALUES ('PASTE-UUID-HERE', 'Admin', 'admin');
Manage Users
  • Supabase account created
  • Project "thelifeco-portal" created
  • API keys copied (URL, anon key, service role key)
  • schema.sql executed successfully
  • seed.sql executed successfully
  • Admin user created and added to staff table
4
Vercel — Hosting & API

Vercel hosts the app and runs the serverless API functions.

4.1 Create Vercel Account Free — Hobby plan

  1. Go to Vercel and sign up with your GitHub account (recommended)
  2. This automatically connects Vercel to your GitHub repos
Sign up for Vercel

4.2 Import GitHub Repository

  1. Click Add New... → Project
  2. Find and select the repo: ibak-app/thelifeco-app
  3. Framework preset: Other
  4. Root directory: app (the app lives inside the app/ folder)
  5. Don't deploy yet — set environment variables first
Import Project

4.3 Set Environment Variables Required

In the project settings, go to Settings → Environment Variables and add:

NameValueEnvironment
SUPABASE_URLhttps://xxxxx.supabase.coAll
SUPABASE_ANON_KEYeyJhbG... (from Phase 3)All
SUPABASE_SERVICE_ROLE_KEYeyJhbG... (from Phase 3)All
RESEND_API_KEYre_xxx... (from Phase 2)All
Vercel Dashboard

4.4 Deploy

  1. After setting env vars, click Deploy
  2. Vercel will build and deploy the app
  3. You'll get a preview URL like thelifeco-app-xxxx.vercel.app
  4. Test everything works on the preview URL before switching DNS

4.5 Add Custom Domain

  1. Go to project Settings → Domains
  2. Add: thelifeco.app
  3. Vercel will tell you to add a DNS record in Cloudflare:

Verification — Test Everything

Confirm all services are connected and working correctly.

  • Email outbound: Claude Code will test sending via Resend API
  • Guest portal: Visit thelifeco.app/guest/hasan-akkaya/ — should load and show PIN screen
  • PIN auth: Enter 0309 — should unlock the app
  • All 4 tabs work: Today, Schedule, Explore, Info
  • Admin login: Visit thelifeco.app/admin/ — login with admin@thelifeco.app
  • Create guest: Create a test guest in admin — their portal URL should work
  • WhatsApp: Tap the WhatsApp button — should open chat
  • PWA install: On mobile, "Add to Home Screen" should work
  • Offline: Enable airplane mode — cached data should still show
